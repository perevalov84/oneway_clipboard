## Описание
Реализация одностороннего "буфера обмена" мжеду rdp клиентом и сервером
при отключеном буфере обмена на сервере политиками GPO.

Передача буфера обмена организована следующим образом:
- На третьем linux сервере устанавливается oneway-clipboard сервис
- Клиент через powershell скрипт отправляет серверу файлы и содерижмое буфера обмена
- Сервис согласно настройкам data.json определяет отправителя по исходящему ip и
передаёт в сессию пользователя на RDP сервере используя и copyq.

## data/data.json
Файл отвечает за споставление отправителя содержимого буфера с учетной запиью
получателя этого буфера обмена.
```
[
    {
        "src_ip": "127.0.0.1",
        "username": "user",
        "dst_ip": "192.168.122.1",
        "pubkey": "user.pub",
        "privkey": "user"
    },
    {
        "src_ip": "10.0.0.68",
        "username": "user",
        "dst_ip": "192.168.122.1",
        "pubkey": "user.pub",
        "privkey": "user"
    }
]

```

## Порядок использования
1. На windows server поднять ssh server
- Включить аутентификацию по ключу в `C:\ProgramData\ssh\sshd_config`

2. Создать ssh ключи для каждого из пользователей
и поместить их в `data/.ssh/`.

3. В домашенем каталоге windows для каждого пользователя создать `.ssh\authorized_keys`
куда поместить соотвествующий публичный ключ полученый на шаге 2
